## **2. Tool Description**
### **2.1 Tool Overview**
`lcov` is a **GNU-based coverage analysis tool** that:
- Parses **gcov** data files (`*.gcda`, `*.gcno`) generated by GCC/Clang.
- Produces **HTML/XML reports** for manual review or automated checks.
- Supports **branch, line, and function coverage**.

### **2.2 Intended Use**
| **Use Case**               | **ISO 26262 ASIL Requirement** |
|----------------------------|--------------------------------|
| Coverage measurement       | ISO 26262-6 (Software Unit Testing) |
| Compliance with MC/DC      | ISO 26262-8 (Modified Condition/Decision Coverage) |
| Automated test validation  | ISO 26262-5 (Verification)     |

### **2.3 Tool Configuration**
- **Input:** GCC-generated coverage data (`gcov`).
- **Output:** HTML/XML reports, summary logs.
- **Dependencies:**
  - GCC/Clang (≥ vX.Y for compatibility)
  - Python (if using `genhtml` wrapper)
  - Operating System: Linux (tested on Ubuntu/Debian)

## **5. Qualification Activities**
### **5.1 Static Analysis (Source Code Review)**
- **Objective:** Identify safety-critical behaviors in `lcov` source code.
- **Method:**
  - Review **C/C++ source** for buffer overflows, race conditions.
  - Check **configuration files** (`--rc`, `--include`, `--exclude`) for safe defaults.
- **Tools Used:**
  - **Clang-Tidy**, **Cppcheck**
  - Manual review by safety experts.

### **5.2 Dynamic Testing**
| **Test Case**               | **Description** | **Expected Result** |
|-----------------------------|-----------------|----------------------|
| **Basic Coverage Calculation** | Run on a simple C program with known coverage. | Correct % reported. |
| **Branch Coverage**         | Test MC/DC compliance (e.g., `if-else` chains). | All branches accounted for. |
| **Edge Cases**              | Empty files, malformed `.gcda`. | Graceful error handling. |
| **Regression Testing**      | Compare against previous versions. | No functional degradation. |

### **5.3 Configuration Control**
- **Version Locking:**
  - Use `apt pinning` (Debian) or `yum versionlock` to prevent upgrades.
  - Example:
